3. CORE FUNCTIONAL MODULES
3.1 Registration & Authentication
•	Dynamic Registration: Users must select their Department from a list managed by the Admin.
•	Fields: Full Name, Email (Unique), Phone, Scientific Degree (PhD/DSc/None), Position.
•	Security: Password hashing via bcrypt. JWT for session persistence.
3.2 Dynamic Submission Wizard (4 Steps)
•	Step 1: Metadata (Title, Abstract, Keywords).
•	Step 2: Classification (Department + Scientific Direction).
•	Step 3: Literature Type (Textbook, Monograph, Study Guide, etc.).
•	Step 4: Document Upload (Max 100MB). Files required:
o	Internal Review, External Review, Plagiarism Report, Curriculum, Syllabus, Main Manuscript.
3.3 Expert Evaluation Logic
Reviewers grade assigned manuscripts on a 1–10 scale for:
•	Scientific Significance, Methodology, Structure, Originality.
•	Final Recommendation: Accept, Minor Revision, Major Revision, Reject.
3.4 System Admin "Dynamic" Dashboard
The Admin can edit the following without touching code:
•	Department Dictionary: (e.g., Adding Амалий информатика, Банк иши, etc.).
•	Scientific Directions: (e.g., Engineering, Medicine, Arts).
•	Submission Constants: Change max file size or required document lists.

4. ON-PREMISE STORAGE & SECURITY
•	File Storage Strategy: Files are saved to the server's local disk. The database stores the file path, not the file itself.
•	Access Control: Users can only download files they are assigned to. The backend uses a "Stream Controller" to verify permissions before sending the file.
•	Virus Scanning: All incoming files are piped through ClamAV before the final save.
•	Automated Backup: A daily cron job performs a pg_dump of the database and an rsync of the file directory to a secondary internal drive.

5. DATABASE ARCHITECTURE (SQL SCRIPT)
You can run this directly on your PostgreSQL server to initialize the portal.
SQL
-- 1. Departments (Managed by Admin)
CREATE TABLE departments (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE
);

-- 2. User Accounts
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    full_name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    role VARCHAR(50) CHECK (role IN ('Author', 'Editor', 'Reviewer', 'Admin')),
    department_id INT REFERENCES departments(id),
    degree VARCHAR(50),
    position VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 3. Submissions
CREATE TABLE submissions (
    id SERIAL PRIMARY KEY,
    title VARCHAR(500) NOT NULL,
    abstract TEXT,
    author_id INT REFERENCES users(id),
    direction VARCHAR(100),
    literature_type VARCHAR(100),
    status VARCHAR(50) DEFAULT 'Submitted',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 4. File Registry (Local Path Pointers)
CREATE TABLE submission_files (
    id SERIAL PRIMARY KEY,
    submission_id INT REFERENCES submissions(id),
    file_type VARCHAR(100),
    file_path TEXT NOT NULL,
    file_size_mb DECIMAL(10,2),
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 5. Reviews & Scores
CREATE TABLE reviews (
    id SERIAL PRIMARY KEY,
    submission_id INT REFERENCES submissions(id),
    reviewer_id INT REFERENCES users(id),
    score_significance INT,
    score_methodology INT,
    score_originality INT,
    verdict VARCHAR(50),
    comments_author TEXT,
    comments_editor TEXT,
    submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 6. Audit Logs
CREATE TABLE audit_logs (
    id SERIAL PRIMARY KEY,
    user_id INT,
    action VARCHAR(255),
    ip_address VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
